# Rules

## General Rules

1. **No exploiting or cheating.**  
        Exploiting refers to using bugs or design flaws to gain an unintended advantage. Examples include:
        - Using x-ray or x-ray texture packs.
        - Creating machines that generate gold.
        - Attempting to bypass the server's Anti-AFK system.
        - Griefing areas that would normally be protected if the server's plot protection fails.

2. **No unauthorized software or modifications.**  
        Avoid using third-party clients, mods, scripts, or macros that provide an unfair advantage in PvP. Examples include:
        - Map icons, nameplate mods, fly mods, no-fall damage mods.
        - Automation tools like aimbots, auto-consume, auto-replace hotbars, auto-fishing, etc.  
        *Note:* Simple one-liner macros (e.g., calling guards, casting spells, repair macros) are allowed. If unsure about a macro, ask an admin. Macro mods are acceptable if not used for complex automation.

3. **Griefing is allowed outside protected plots.**  
        Griefing is permitted outside of other players' plots. However, griefing near admin-owned towns may be cleared to maintain playability.

4. **Respect privacy and personal boundaries.**  
        - Do not share or investigate personal real-life information about other players (e.g., names, locations).  
        - Keep insults limited to in-game matters. Avoid topics like race, religion, sexual orientation, or mental health. Attack the character, not the player.

5. **No admin impersonation.**  
        Do not claim to be an admin, falsely identify others as admins, or pretend to have admin powers.

6. **Report suspected cheating responsibly.**  
        If you suspect someone is cheating, use `/report <minecraft name> <reason>` or DM an admin on Discord. Avoid public accusations.

7. **Do not disrupt admin operations.**  
        This includes:
        - Disrespecting admins or evading bans.
        - Lying to admins or creating unresolvable traps.
        - Building machines or clocks that cause server stress (machines are fine if actively attended and used for a purpose).  
        *Note:* This is a catch-all rule to ensure smooth server operations.
